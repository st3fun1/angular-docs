Observer is an object that monitors a stream and responds to notifications represented as methods.

It uses the following methods: .next(), .error(), .complete

An observer is a collection of callbacks that knows how to listen to values delivered by the Observable.

Subscriber is an observer which can unusbscribe from an Observable.

An Observable stream is any stream of data produced(emited) overtime:

  • Numbers
  • Streams
  • Events
  • Http Response
  • Other Observable streams

Observables can be sync or async

Can emit a finite or infinite number of data

Observables are lazy and doesn't emit any data, until you subscribe to the stream

To start start an Observable stream you must subscribe to it using .subscribe()

When we subscribe we pass an onbserver to monitor a subscription.

As part of the subscription process, we tell the Observable how we gonna observe it.

Observer - set of callbacks to observe, handling next, error and complete

Observable or stream is a sequence of emitted items

Ways to stop an Observable Stream

  • Call complete
  • User completing operator which unsubscribe
  • Throw an error
  • Calling .unsubscribe on subscription

Why should we do this?

Unsubscribing from each Observable prevents memory leaks

Creating an Observable

One of the following creation functions can be used

  • .of
  • .from
  • .fromEvent creates an Observable from any DOM event
  • .interval creates an Observable that emits a seq number at a defined interval of time

.of vs .from

Given an array of data, the stream behaves the following:

.from takes each element in the structure and emits it

.of emits all the items in the structure as a whole
