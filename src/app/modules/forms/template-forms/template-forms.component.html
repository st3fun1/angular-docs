Inside a template driven form the control elements are bound to data properties, with input validation to maintain data integrity
and styling to improve the user experience.

Template-driven forms are using two-way data binding to update the data model in the component as changes are made in the template and vice versa.

Are suitable for small or simple forms

Form control elements should be named with "name"

Each for control can be tracked and you can see its states

  • The control has been visited
    • class if true -> ng-touched
    • class if false -> ng-untouched
  • The control's value has changed
    • class if true -> ng-dirty
    • class if false -> ng-pristine
  • The control's value is valid
    • class if true -> ng-valid
    • class if false -> ng-invalid

Angular can reference the input with a template reference variable, the value is ngModel because that is the value of NgModel.exportAs property. This property tells Angular how to link a reference variable to a directive.
