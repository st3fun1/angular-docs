What is RxJS?

Is a declarative programming paradigm concerned with data streams and the propagation of change.

Reactive programming is functional.

Reactive Development

  • Quick to react to user interactions
  • Resilient to failure. Good error handling
  • Reactive to state changes. Code watches for change events

Data Stream

Data is not continuous like an actual stream. You actually need to subscribe to the stream.

Processing Observable Streams

  1. Start the stream. Emit items into the stream.
  2. Items pass through a set of operations.
  3. As an observer:
    • Pick next item and process it
    • When an error occurs, handle it
    • When there's nothing left to process, you'll receive a notification that you're done
  4. If you need to leave, stop the stream
