View Encapsulation

View Encapsulation controls whether we use emulate, we use Shadow DOM or we don't use styling scoping behavior.

View Encapsulation Modes

  1. None

    Completely turns off Emulated Shadow DOM encapsulation and puts all the styling in the head, unscoped.

    Use cases: In App component(root), for global styling.

  2. Emulated(Default)

    By default, Angular, add some scoping attributes to a node within a component.

    The custom element gets _nghost custom attribute

    The content from the template gets _ngcontent

    How does Angular processes the CSS?

    If View Encapsulation is Emulated they are put in the style tag in head, with a combination of selector + custom attribute

  3. Native(Shadow DOM)

    It uses Native Shadow DOM

    An actual shadow root will be created
